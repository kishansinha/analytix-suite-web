🛠️ The Project

Analytix Suite is a growing collection of statistical analysis tools tailored specifically for teachers and school-based action researchers. Each module addresses a different analytical need — from basic test performance summaries to advanced correlation and regression analysis — all without requiring any knowledge of statistics software or coding.

Every module is free to use. Your data is never stored. The tools are designed to be practical, interpretable, and immediately actionable in a classroom context.

How to Use

Get up and running with Analytix Suite in a few simple steps.

1

Download the Excel Template

Go to the Downloads tab and download the Analytix data template. This is the standard format all modules expect. Enter your student data — question numbers, AO codes, topic labels, marks allocated, and student scores — into the template.

2

Choose a Module

Return to Home and select the module that fits your analytical goal. If you're unsure, start with Single Test Analysis (Basic) or Descriptive Statistics (Advanced) for a broad overview of your data.

3

Upload Your File

Each module has a file upload area. Upload your completed Excel template. Modules that support cross-test analysis allow you to upload multiple files at once.

4

Configure & Run

Select the variables, statistical options, and chart types you want. Then click Run Analysis. The server processes your data and prepares your results — typically within a few seconds.

5

Download Your Results

Results are packaged as a ZIP file containing charts (PNG) and a formatted Excel summary report. Download, open, and explore — no specialist software required.

💡 New to action research? Start with the Single Test Analysis module and a single test file. Once you're comfortable with the output format, move to multi-test modules like Progress Tracking or Correlation Analysis.

Analytix Data Template Required

The standard Excel template used by all Analytix modules. Enter student names, question numbers, AO codes, topic labels, marks allocated, and student scores. Compatible with every module in the suite.

⬇ Download Template (.xlsx) ⬇ Download AO Template (.xlsx)
📐 File Structure

The template uses one row per question and one column per student. Row 2 contains column headers — replace "Student 1", "Student 2", etc. with actual student names. Questions are listed from Row 3 downwards.

🗂️ Columns
Col A
Question

One question per row. Use the format Paper_Question — e.g. 1A_7 (Question 7, Paper 1A) or 2_3(a)(iii) (Question 3(a)(iii), Paper 2). For a single paper, plain numbering like 7 or 3(a)(iii) also works.

Col B
Topic

Topic name for each question. Used to group questions in topic-level charts.

Col C
AO (primary)

AO code for each question, e.g. AO2j. Use codes from the AOs File.

Col D
AO (secondary) — reserved

Leave blank. Reserved for future analysis. Entries here will not affect results.

Col E
Marks Allocated

Marks available for each question. This is the only marks column used in analysis.

Col F
Marks per Question — optional

You may use this to record total marks for a main question (e.g. the sum across sub-parts). Not used in analysis.

Col G+
Student score columns

One column per student. Replace "Student 1", "Student 2", etc. in Row 2 with real names. Enter each student's score for each question.

⚠️ Special Values
🔴
Absent students: Insert AB in the first data cell (Row 3) below the student's name. Leave all other cells for that student blank.
🟡
Unattempted questions: Enter NA — not 0. An unattempted question (lack of time, unfamiliar context) has different meaning from an incorrectly attempted one.
💡
Multiple papers: Combine all papers into one file using the Paper_Question format, or maintain a separate file per paper — both are supported.
📐 File Structure

The AOs File is a simple two-column reference list — it contains no question mapping. Column A holds the AO code (e.g. AO2j) and Column B holds the descriptor (the full description of what that AO assesses).

🗂️ Columns
Col A
Code

The AO code used in the Data Template (Col C). These must match exactly — the module uses these codes to link questions to their objectives.

Col B
Assessment Objective

A plain-language description of the skill or objective. Used as labels in charts and reports.

✏️ Customising Your AOs

The provided AOs File is based on a standard set of assessment objectives. You have three options:

✅
Use as-is: If the provided AO codes match your curriculum, simply use the file without changes.
✏️
Modify: Edit the codes and descriptors to match your school's or curriculum's AO framework. Ensure the codes in Col A match what you enter in Col C of the Data Template.
🆕
Create your own: Replace all rows with your own AO codes and descriptors. Any consistent coding system works — e.g. K1, K2 for knowledge objectives, A1, A2 for application, etc.

About How to Use Contact Support Terms
📋

Descriptive Statistics

Summarise class scores, compute means, medians, spread, and generate visual frequency distributions. The ideal starting point for any action research project.

📊

Single Test Analysis

Upload results from a single assessment and receive a full breakdown of performance, item difficulty, and question-level insights.

📈

Progress Tracking

Compare results across two or more time points to identify growth patterns, find students who need support, and visualise class-wide progress.

📊

Comparative Analysis

Run t-tests and ANOVA to compare groups — useful for evaluating whether an intervention produced a statistically significant difference in outcomes.

🔗

Correlation Analysis

Explore relationships between variables — attendance and performance, prior attainment and progress, effort ratings and outcomes.

📉

Regression Analysis

Build predictive models from your classroom data and detect outliers — students whose performance diverges significantly from what the model would predict.
