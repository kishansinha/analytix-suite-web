MOD-01 How to Use
📋 What This Module Does

Single Test Analysis takes results from one assessment and generates two sets of output charts: individual student reports and class-wide performance charts. Individual reports break down each student's performance by AO category, specific AO, and topic — ideal for sharing directly with students as personalised, actionable feedback. Class charts reveal patterns, strengths, and areas for improvement across the whole cohort.

📁 Files You Need
1
Data File — Analytix_Data_Template_File.xlsx

Download the Analytix Data Template from the Analytix Suite home page (Downloads tab). Each row represents one question. Each student occupies one column — replace "Student 1", "Student 2", etc. with actual student names in Row 2.

2
AOs File — AOs_Template_File.xlsx

Download the AOs File from the Downloads tab. It contains a list of AO codes and their descriptors. You can use the provided AOs as-is, modify them, or replace them with your own set of Assessment Objectives.

ℹ️

Both files are available in the Downloads tab on the Analytix Suite home page.

📝 Filling In the Data Template
A
Question column (Col A)

Enter one question per row. Use the format Paper_Question — for example, 1A_7 means Question 7 in Paper 1A, and 2_3(a)(iii) means Question 3(a)(iii) in Paper 2. If you are using a single paper or prefer simple numbering, you can use plain numbers like 7 or 3(a)(iii).

B
Topic column (Col B)

Enter the topic name for each question. This is used to group questions in the topic-level charts.

C
AO columns (Col C & D)

Enter the AO code for each question in Column C (e.g. AO2j). Column D is reserved for future analysis — leave it blank. Entering a code in Column D will not affect results.

D
Marks Allocated column (Col E)

Enter the marks available for each question. This is the only marks column used in analysis.

E
Marks per Question column (Col F)

Optional. You may use this to record total marks for main questions (e.g. the total for a multi-part question). It is not used in analysis.

F
Student score columns (Col G onwards)

Replace "Student 1", "Student 2", etc. in Row 2 with actual student names. Enter each student's score for each question in the corresponding cell. Use whole numbers or decimals as appropriate.

⚠️

Absent students: Insert AB in the first data row below the student's name. Do not enter scores for absent students.

⚠️

Unattempted questions: Enter NA — not 0. An unattempted question (lack of time, unfamiliar context) carries different meaning from an incorrectly attempted one.

🔢 Step-by-Step
1
Upload your files (Panel 1)

Drag and drop or click to upload your Data File and AOs File. The module will validate both files and show a summary of detected students, questions, and total marks. Fix any errors flagged before proceeding.

2
Configure your analysis (Panel 2)

Enter the exam or grade label used in charts. Choose whether to run Individual Reports, Class Analysis, or both, and select your preferred chart options.

3
Generate Analysis

Click Generate Analysis. Panel 3 shows a live progress log as the server processes your data. Typical run time is 15–45 seconds depending on class size.

4
Download your results (Panel 3)

When complete, click Download ZIP. The ZIP contains all charts as PNG files. Individual student charts are in a separate subfolder — ready to share directly with students as feedback.

📊 What You Get
👤
Individual Student Reports

Bar charts showing each student's percentage score broken down by AO category, specific AO, and topic. One chart set per student — shareable directly as personalised feedback.

🌡️
Class Heatmap

A grid showing every student's performance across every AO and topic. Instantly reveals which skills the class struggled with and which individuals are outliers.

🎻
Violin / Distribution Plots

Show the spread and shape of scores across AO categories and topics. Reveals whether performance was consistent or polarised within each skill area and topic.

🕸️
Radar Chart

Class average performance across all AO categories and topics on a single chart — useful for identifying overall strengths and gaps at a glance.

💡 Tips
💡

Sharing with students: Individual report charts are named using the student's column header. Use consistent, full names so files are easy to identify and distribute.

💡

Multiple papers: You can combine all papers into one Data File using the Paper_Question format, or maintain a separate file per paper — both approaches work.

ℹ️

Your data is never stored. Files are processed on the server and deleted immediately after your ZIP is generated.
