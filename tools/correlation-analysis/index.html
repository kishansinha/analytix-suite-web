MOD-04 How to Use
📋 What This Module Does

Correlation Analysis examines statistical relationships between pairs of variables in your test data — overall scores, Assessment Objectives, and topics. Upload one or more Data Files, choose an analysis mode, select your variable pairs, configure the statistical tests and charts you want, and the module produces a comprehensive ZIP containing PNG charts and a multi-sheet Excel summary.

ℹ️

No AOs File needed. Overall scores, AOs, and topics are detected automatically from your Data File(s). Absent students (marked AB) are detected and excluded before any analysis begins.

💡

Run the Descriptive Statistics module (MOD-00) first — its test recommendations will tell you which correlation tests and corrections are most appropriate for your data.

📁 Files You Need
1
One or more Data Files — Analytix_Data_Template_File.xlsx

Your filled-in Analytix Data Template(s). For Single Test mode, upload exactly one file. For all other modes, upload two or more. Download the template from the Downloads tab on the Analytix Suite home page.

🔢 Step-by-Step
1
Choose your analysis mode first (Panel 1)

Select the mode that fits your research question before uploading files. The mode determines how many files the module will accept: Single Test locks the upload to one file; all other modes accept two or more. See the Analysis Modes section below for guidance on which to choose.

2
Upload and load your Data File(s) (Panel 1)

Drag and drop or click to upload your .xlsx file(s). Click Load & Detect Variables. The module reads all files, excludes absent students, and detects all variables available for analysis. A status summary shows how many students, AOs, and topics were found in each file.

3
Build your variable pairs (Panel 2)

Select Variable A and Variable B from the dropdowns and click Add This Pair. Repeat for each pair. Use the Quick Add shortcuts to add all Overall ↔ AO pairs, all AO ↔ AO pairs, or all Topic ↔ Topic pairs in one click. Click Proceed to Run when your pairs are set.

4
Configure tests, options, and charts (Panel 3)

Choose which correlation statistics to compute, set filters, select charts, and adjust advanced options. Confidence intervals are enabled by default. Click Run Analysis when ready.

5
Download your results (Panel 3)

When the run completes, click Download ZIP. The archive contains all charts as PNG files and the Correlation Analysis Summary.xlsx with all results in separate named sheets.

⚙️ Analysis Modes

Select the mode in Panel 1 before uploading files — it controls how many files are accepted and how data is combined.

1️⃣
Single Test

Correlates variables within a single test file. Use when you want to understand how AOs or topics relate to each other or to overall performance within one assessment. Accepts one file only.

🔁
Multiple Tests — between tests

Correlates variables across two or more tests — e.g. does AO2 in Test 1 predict AO2 in Test 2? Variables from each file are prefixed (T1_, T2_) so you can select cross-test pairs directly. Requires two or more files. Time Series analysis is only available in this mode.

📊
Pooled — Averaged (n = students)

Averages each student's scores across all uploaded tests, then computes correlations on the averaged values. The unit of analysis is the student (n = number of unique students). Good for reducing noise from individual test variation when you have multiple tests of the same cohort.

🔢
Pooled — Independent (n = observations)

Treats each student–test pair as a separate data point (e.g. Saksham in Test 1 and Saksham in Test 2 are two independent observations). Increases sample size but assumes independence between rows — use for exploratory work. Note: pairs involving an AO that only appears in some tests will have a smaller n than pairs spanning all tests; these are flagged in the Excel output.

🧮 Statistics & Options
📐
Correlation tests

Pearson r is always computed (parametric, assumes linearity and approximate normality). Spearman ρ is rank-based and robust to non-normal data — preferred when Assumption Checks flag non-normality. Kendall τ is more robust for small samples (n < 20). Percentage Bend is a robust alternative when outliers are present.

📏
Confidence Intervals (on by default)

95% Fisher's z confidence intervals are computed automatically for every pair. Choose 90%, 95%, or 99%. Enable Bootstrap CI for more accurate intervals when data is non-normal (slower). Wide CIs (width > 0.5) are flagged in orange in the Excel — they indicate the correlation estimate is imprecise, typically due to small n.

🔍
Assumption Checks

Runs Shapiro-Wilk normality tests and linearity checks (Pearson vs Spearman gap) for every pair. Results appear in the Assumption Checks sheet of the Excel and as Q-Q + residual + distribution charts. If normality fails, the Excel recommendation suggests switching to Spearman or Bootstrap CI.

🧩
Partial Correlations

Computes the correlation between two variables after removing the influence of a control variable. Two strategies: Control for Overall Score only (asks whether an AO–AO relationship is real or just reflects general ability) or Control for all other selected variables. Results appear in the Partial Correlations sheet, showing partial r, raw r, and the change — large drops indicate the relationship was largely explained by the control.

🎛️
Filters & Multiple Comparison Correction

Filter to show only significant pairs (p < 0.05) or by minimum effect size. When analysing many pairs, apply multiple comparison correction: Bonferroni is more conservative; Benjamini-Hochberg (FDR) is preferred when you have a large number of pairs and want to limit false discoveries without being too strict.

🔎
Outlier Detection

Flags data points with z-scores exceeding a threshold (default 3.0). For every pair with outliers, computes r both with and without those points (Δr) and generates a comparison chart. The Outlier Comparison and Outlier Detail sheets list exactly which students were flagged and why.

💡

Not sure which tests to use? Enable Assumption Checks on your first run — the recommendations in the Assumption Checks sheet will tell you whether Pearson or Spearman is appropriate for each pair.

📊 Reading the Excel Output
📋
Overview sheet

Summary of the run: tests analysed, mode used, any absent students excluded, total pairs computed, and a statistical power summary. When pair sample sizes differ (common in Pooled Independent mode), the n range and a note pointing to the Correlations sheet are shown instead of a single value.

📈
Correlations sheet — colour coding

The Pearson r cell is colour-coded by both strength and direction: green = strong positive, red-pink = strong negative, yellow/amber = medium, light amber = small. The n cell is highlighted amber when n < 15 — treat those correlations with caution. The CI cells appear in orange when the confidence interval is wide (width > 0.5).

⚡
Min Detectable |r| column

The smallest effect size this pair had 80% power to detect at α = 0.05, given its n. Each pair has its own value because n varies between pairs. A high minimum detectable r (e.g. ≥ 0.70) means only large effects can be reliably found — smaller effects may be real but undetectable at this sample size.

🧑‍🎓
Student Profiles sheet

One row per student, one column per selected variable. Cells are shaded by within-variable performance quartile: red (bottom 25%), amber, yellow, green (top 25%). Useful for identifying students with uneven AO profiles or consistently weak areas across all variables.

🗂️
Raw Data sheet

Shows the exact values used to compute each correlation — one column pair per variable pair. Lets you verify that the right students and scores were included and check for any unexpected values.

🔒
Reliability sheet

Cronbach's α (computed on aggregated AO/topic scores) and Split-Half Spearman-Brown reliability (computed on raw question scores, averaged over 20 random splits). The two methods are independent — Cronbach's α measures internal consistency across AOs; Split-Half measures consistency across individual questions. Both are shown with an interpretation label (Excellent / Good / Acceptable / Questionable / Poor).

🎯
Predictive Validity sheet

For multi-test runs: how well does performance on a variable in Test i predict performance on the same variable in Test i+1? Higher r² means stronger predictive power. Useful for identifying which AOs show consistent student rankings over time.

📅
Time Series sheet

Available in Multiple Tests mode only. Tracks how the correlation of each variable with itself changes across consecutive test pairs (stability). A high r means student rankings on that variable are consistent from test to test.

💡 Tips & Cautions
💡

Start broad, then drill down. Use Quick Add (Overall ↔ AO pairs) on your first run to identify the strongest relationships. Then re-run with only those pairs and Assumption Checks enabled to get rigorous results.

💡

Partial Correlations reveal what's really going on. A strong AO1–AO2 correlation often disappears after controlling for Overall Score — meaning it just reflects general ability, not a genuine AO1–AO2 relationship. Use partial correlations before drawing conclusions about which specific skills are related.

💡

Apply multiple comparison correction when analysing many pairs. With 30+ pairs, you expect 1–2 spurious significant results by chance at α = 0.05. Benjamini-Hochberg (FDR) is generally preferred over Bonferroni for a large number of pairs.

💡

Small n means wide CIs. With 13–27 students, confidence intervals around r are typically ±0.3 or wider. A result of r = 0.65 might have a 95% CI of [0.28, 0.85] — the true value could be anywhere in that range. Wide CI cells are flagged in orange to remind you of this.

ℹ️

Time Series requires Multiple Tests mode and two or more files. It is automatically disabled when Single Test mode is selected.

ℹ️

Your data is never stored. Files are processed on the server and deleted immediately after your ZIP is generated.
